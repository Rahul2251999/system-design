Network Basics

Beginner 15 min read

Introduction to Computer Networks

Computer networks are the backbone of modern distributed systems, enabling communication between different components and services. Understanding networking fundamentals is crucial for effective system design, as network characteristics significantly impact system performance, reliability, and scalability.

A computer network is a collection of computing devices connected together to share resources and communicate with each other. These connections can be physical (wired) or wireless, and can span anything from a small local area to the entire globe.

The OSI Model

The Open Systems Interconnection (OSI) model is a conceptual framework that standardizes the functions of a telecommunication or computing system into seven abstraction layers. Understanding this model helps in grasping how different networking technologies interact and where specific protocols operate.

Layer Name Function Examples
7 Application Provides network services directly to end-users or applications HTTP, SMTP, FTP, DNS
6 Presentation Translates data between the application layer and the network format SSL/TLS, JPEG, MPEG
5 Session Manages communication sessions between applications NetBIOS, RPC
4 Transport Provides reliable data transfer between end systems TCP, UDP
3 Network Routes data packets between different networks IP, ICMP, OSPF
2 Data Link Transfers data between adjacent network nodes Ethernet, Wi-Fi, PPP
1 Physical Transmits raw bit stream over physical medium Cables, switches, hubs

In system design, we primarily focus on the upper layers (especially Application, Transport, and Network), as these have the most direct impact on application behavior and performance.

Key Networking Concepts

IP Addressing

IP (Internet Protocol) addresses are unique identifiers assigned to devices on a network. They come in two main versions:

IPv4

The older standard using 32-bit addresses (e.g., 192.168.1.1). With only about 4.3 billion possible addresses, IPv4 addresses are becoming scarce.

IPv6

The newer standard using 128-bit addresses (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334). IPv6 provides a vastly larger address space to accommodate the growing number of internet-connected devices.

IP addresses can be:

  • Public: Globally routable addresses visible on the internet
  • Private: Used within local networks and not directly accessible from the internet
  • Static: Permanently assigned to a device
  • Dynamic: Temporarily assigned by a DHCP server

Ports

While IP addresses identify devices on a network, ports identify specific applications or services running on those devices. Ports are 16-bit numbers (0-65535) that enable multiple network services to operate on a single device.

Common port numbers include:

  • 80: HTTP (web)
  • 443: HTTPS (secure web)
  • 22: SSH (secure shell)
  • 25: SMTP (email sending)
  • 3306: MySQL database
  • 27017: MongoDB database

DNS (Domain Name System)

DNS translates human-readable domain names (like example.com) into IP addresses that computers use to identify each other. This hierarchical, distributed system is crucial for the usability of the internet and networked applications.

1

DNS Query

A client requests the IP address for a domain name from a DNS resolver.

2

Recursive Resolution

The resolver queries root servers, then TLD servers, then authoritative name servers.

3

Response

The IP address is returned to the client, often with caching at various levels to improve performance.

DNS is a critical component in system design, affecting everything from service discovery to load balancing strategies.

Transport Protocols

Transport protocols operate at layer 4 of the OSI model and are responsible for end-to-end communication between devices. The two primary transport protocols are TCP and UDP, each with distinct characteristics that make them suitable for different use cases.

TCP (Transmission Control Protocol)

TCP is a connection-oriented protocol that provides reliable, ordered, and error-checked delivery of data. It establishes a connection before data transfer and ensures all data arrives correctly.

TCP Characteristics

  • Connection-oriented: Establishes a connection before data transfer
  • Reliable: Guarantees delivery of data
  • Ordered: Data arrives in the order it was sent
  • Flow control: Prevents overwhelming receivers with too much data
  • Congestion control: Adapts to network congestion
  • Error detection: Identifies and retransmits lost or corrupted data

Use cases: Web browsing (HTTP/HTTPS), email, file transfers, and any application requiring reliable data delivery.

UDP (User Datagram Protocol)

UDP is a connectionless protocol that provides a simple, unreliable datagram service. It sends data without establishing a connection and does not guarantee delivery or ordering.

UDP Characteristics

  • Connectionless: No connection establishment
  • Unreliable: No guarantee of delivery
  • Unordered: Data may arrive in a different order than sent
  • No flow control: Sends data regardless of receiver capacity
  • No congestion control: Doesn't adapt to network conditions
  • Lightweight: Lower overhead than TCP

Use cases: Real-time applications like video streaming, VoIP, online gaming, and DNS lookups where speed is more important than reliability.

Feature TCP UDP
Connection Connection-oriented Connectionless
Reliability Reliable Unreliable
Ordering Ordered delivery No ordering
Speed Slower Faster
Overhead Higher Lower
Use Cases Web, email, file transfer Streaming, gaming, VoIP

Application Layer Protocols

Application layer protocols operate at the highest level of the OSI model and provide specific services to applications. Here are some key protocols relevant to system design:

HTTP/HTTPS (Hypertext Transfer Protocol)

HTTP is the foundation of data communication on the World Wide Web. It defines how messages are formatted and transmitted, and how web servers and browsers should respond to various commands.

HTTP Characteristics

  • Request-response model: Client sends requests, server sends responses
  • Stateless: Each request is independent
  • Methods: GET, POST, PUT, DELETE, etc.
  • Status codes: 200 (OK), 404 (Not Found), 500 (Server Error), etc.
  • Headers: Metadata about the request or response

HTTPS is the secure version of HTTP, using TLS/SSL encryption to protect data in transit.

WebSocket

WebSocket provides full-duplex communication channels over a single TCP connection, enabling real-time, bidirectional communication between clients and servers.

WebSocket Characteristics

  • Persistent connection: Stays open until explicitly closed
  • Bidirectional: Both client and server can send messages at any time
  • Low latency: Ideal for real-time applications
  • Efficient: Lower overhead than repeated HTTP requests

gRPC

gRPC is a high-performance, open-source RPC (Remote Procedure Call) framework that can run in any environment. It uses Protocol Buffers as the interface definition language and supports multiple programming languages.

gRPC Characteristics

  • Strongly typed: Uses Protocol Buffers for service definitions
  • Efficient serialization: Binary format instead of text
  • Multiple patterns: Supports unary, server streaming, client streaming, and bidirectional streaming
  • Built on HTTP/2: Benefits from features like multiplexing and header compression

MQTT (Message Queuing Telemetry Transport)

MQTT is a lightweight publish-subscribe messaging protocol designed for constrained devices and low-bandwidth, high-latency networks, making it ideal for IoT applications.

MQTT Characteristics

  • Publish-subscribe: Publishers send messages to topics, subscribers receive messages from topics they're interested in
  • Lightweight: Minimal packet overhead
  • Quality of Service: Three levels (at most once, at least once, exactly once)
  • Retained messages: Last message can be stored for new subscribers

Network Topologies

Network topology refers to the arrangement of elements in a network. Different topologies have different characteristics that affect reliability, scalability, and performance.

Common Network Topologies

Star Topology

All devices connect to a central hub or switch. Easy to set up and manage, but the central node is a single point of failure.

Ring Topology

Devices connect to exactly two other devices, forming a ring. Data travels around the ring in one direction. Failure of one device can affect the entire network.

Mesh Topology

Devices connect to multiple other devices. Highly reliable due to redundant connections but complex and expensive to implement fully.

Tree Topology

Hierarchical structure with a root node and branches. Scalable and manageable but dependent on higher-level nodes.

Bus Topology

All devices connect to a single backbone cable. Simple and inexpensive but vulnerable to backbone failures.

In modern distributed systems, hybrid topologies combining elements of different basic topologies are common, especially in cloud environments where the physical topology is abstracted away from the application developer.

Network Performance Metrics

Understanding network performance metrics is crucial for designing systems that meet performance requirements. Here are the key metrics to consider:

Bandwidth

Bandwidth refers to the maximum rate of data transfer across a network, typically measured in bits per second (bps), with common units being Mbps (megabits per second) or Gbps (gigabits per second).

Higher bandwidth allows for more data to be transmitted simultaneously, which is important for data-intensive applications like video streaming or large file transfers.

Latency

Latency is the time it takes for data to travel from source to destination, typically measured in milliseconds (ms). It includes:

  • Propagation delay: Time for a signal to travel through the medium
  • Transmission delay: Time to push all the packet's bits into the link
  • Processing delay: Time for routers to process the packet header
  • Queuing delay: Time the packet spends waiting in router queues

Low latency is critical for real-time applications like video conferencing, online gaming, and financial trading systems.

Throughput

Throughput is the actual rate of successful data transfer, which may be lower than the theoretical bandwidth due to various factors like protocol overhead, network congestion, and packet loss.

Packet Loss

Packet loss occurs when data packets fail to reach their destination. It's typically measured as a percentage of packets lost relative to packets sent.

Causes of packet loss include network congestion, hardware failures, and signal degradation. TCP automatically retransmits lost packets, while UDP applications must handle packet loss at the application level.

Jitter

Jitter is the variation in latency over time. High jitter can cause problems for real-time applications, even if average latency is low.

Applications like VoIP and video streaming often use jitter buffers to compensate for variable packet arrival times.

Network Security Concepts

Network security is a critical aspect of system design, protecting data and services from unauthorized access and attacks.

Firewalls

Firewalls monitor and control incoming and outgoing network traffic based on predetermined security rules. They can be:

  • Network firewalls: Operate at the network level, filtering traffic based on IP addresses, ports, and protocols
  • Application firewalls: Operate at the application level, analyzing the content of traffic for specific applications
  • Next-generation firewalls: Combine traditional firewall capabilities with advanced features like intrusion prevention and deep packet inspection

VPN (Virtual Private Network)

VPNs extend private networks across public networks, enabling users to send and receive data as if their devices were directly connected to the private network. They provide:

  • Encryption: Protecting data in transit
  • Authentication: Ensuring only authorized users can access the network
  • Privacy: Hiding the user's IP address and location

TLS/SSL

Transport Layer Security (TLS) and its predecessor, Secure Sockets Layer (SSL), are cryptographic protocols that provide secure communication over a network. They ensure:

  • Privacy: Encrypting data to prevent eavesdropping
  • Integrity: Ensuring data hasn't been modified in transit
  • Authentication: Verifying the identity of communicating parties

HTTPS is HTTP over TLS/SSL, providing secure communication for web applications.

DDoS Protection

Distributed Denial of Service (DDoS) attacks attempt to disrupt normal traffic to a service by overwhelming it with a flood of traffic from multiple sources. Protection strategies include:

  • Traffic filtering: Identifying and blocking malicious traffic
  • Rate limiting: Restricting the number of requests from a single source
  • Anycast network: Distributing traffic across multiple data centers
  • CDN: Using Content Delivery Networks to absorb traffic

Network Considerations in System Design

When designing distributed systems, several network-related factors should be considered:

Geographical Distribution

The physical location of system components affects latency and availability. Options include:

  • Single region: Lower latency within the region but vulnerable to regional outages
  • Multi-region: Higher availability but increased complexity and potential consistency challenges
  • Edge computing: Placing computation closer to users to reduce latency

Network Partitioning

Network partitions occur when network failures prevent nodes from communicating with each other. Systems must be designed to handle these situations gracefully, often involving trade-offs between consistency and availability (as described by the CAP theorem).

Protocol Selection

Choosing appropriate protocols for different components of a system is crucial:

  • User-facing services: HTTP/HTTPS for broad compatibility
  • Internal services: gRPC for efficient service-to-service communication
  • Real-time features: WebSockets or Server-Sent Events
  • IoT devices: Lightweight protocols like MQTT

Network Optimization

Techniques to improve network performance include:

  • Content Delivery Networks (CDNs): Distributing content closer to users
  • Connection pooling: Reusing connections to reduce overhead
  • Compression: Reducing the size of transmitted data
  • Batching: Combining multiple requests or responses
  • Asynchronous communication: Not waiting for responses before continuing processing

Conclusion

Understanding network fundamentals is essential for effective system design. Networks form the foundation upon which distributed systems are built, and their characteristics significantly impact system behavior, performance, and reliability.

Key takeaways from this article include:

  • The OSI model provides a conceptual framework for understanding network layers and protocols
  • TCP and UDP offer different trade-offs between reliability and performance
  • Application layer protocols like HTTP, WebSocket, and gRPC serve different communication needs
  • Network performance metrics (bandwidth, latency, throughput) affect user experience and system capabilities
  • Network security is a critical consideration in system design
  • Geographical distribution, protocol selection, and network optimization techniques can significantly impact system performance

As you design systems, consider network characteristics early in the process, as they will influence many architectural decisions and ultimately determine whether your system can meet its performance and reliability requirements.
